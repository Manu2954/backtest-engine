1. BOOLEAN EXPRESSION LOGIC (MISSING)
-------------------------------------
Backend: Supports named condition groups with boolean expressions
         e.g., "(oversold && trending) || bullish_cross"

Frontend Gap:
  - Only supports single entry/exit condition groups with AND/OR logic
  - No support for named condition groups (entry_groups, exit_groups)
  - No expression editor for complex boolean expressions
  - No support for expression operators (&&, ||, !, parentheses)

Files to modify:
  - types/index.ts: Add entry_groups, exit_groups, entry_expression, exit_expression
  - StrategyBuilder.tsx: Add named group management and expression editor
  - api/client.ts: Update payload structure

Implementation needed:
  a) Add UI to create multiple named condition groups
  b) Add expression editor with syntax highlighting
  c) Add expression validation
  d) Support: && (AND), || (OR), ! (NOT), parentheses


2. DYNAMIC STOP LOSS (MISSING)
------------------------------
Backend: Supports indicator-based trailing stops via dynamic_stop_column

Frontend Gap:
  - No UI to select an indicator column for dynamic stop
  - Can only use fixed percentage stops

Files to modify:
  - types/index.ts: Add dynamic_stop_column to BacktestCreate
  - StrategyBuilder.tsx: Add dynamic stop selector (dropdown of indicator aliases)

Example usage:
  - Stop at "close - 2*ATR" → requires creating indicator column first
  - Stop at Kijun-sen → select ichimoku_kijun column


3. MULTI-COLUMN INDICATOR AWARENESS (INCOMPLETE)
------------------------------------------------
Backend: Multi-column indicators create multiple output columns:
  - MACD → macd_<alias>_macd, macd_<alias>_signal, macd_<alias>_hist
  - BB → bb_<alias>_upper, bb_<alias>_mid, bb_<alias>_lower
  - STOCH → stoch_<alias>_k, stoch_<alias>_d
  - ADX → adx_<alias>, adx_<alias>_dmp, adx_<alias>_dmn
  - ICHIMOKU → <alias>_tenkan, _kijun, _span_a, _span_b, _chikou

Frontend Gap:
  - Condition dropdowns only show the base alias, not sub-columns
  - Users can't easily select macd_signal vs macd_macd
  - No visual indication of which columns each indicator creates

Files to modify:
  - StrategyBuilder.tsx: Expand indicatorAliases to include sub-columns
  - Add helper to generate all column names for multi-output indicators
